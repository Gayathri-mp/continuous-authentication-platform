# YourCredence Deployment Guide (Render)

This guide will help you deploy the **Adaptive Continuous Authentication Platform** to Render for free.

## Prerequisites

1.  A [GitHub](https://github.com/) account.
2.  A [Render](https://render.com/) account (connected to your GitHub).

## Step 1: Prepare Your Repository

1.  Create a new private repository on GitHub.
2.  Push your project code to this repository:
    ```bash
    git init
    git add .
    git commit -m "Prepare for deployment"
    git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
    git push -u origin main
    ```

## Step 2: Deploy to Render

1.  Log in to your [Render Dashboard](https://dashboard.render.com/).
2.  Click **New +** and select **Blueprint**.
3.  Connect your GitHub repository.
4.  Render will automatically detect the `render.yaml` file and show the services to be created.
5.  **Important**: You will need to fill in two environment variables for the `yc_auth_backend` service:
    - `RP_ID`: The domain of your frontend. After the first deploy, this will be something like `yc-auth-frontend.onrender.com`.
    - `RP_ORIGIN`: The full URL of your frontend, e.g., `https://yc-auth-frontend.onrender.com`.
    - _Note: You can use dummy values initially and update them after the frontend URL is assigned by Render._

## Step 3: Architecture in the Cloud

- **Database**: A free PostgreSQL instance stores your users, sessions, and **ML models** (now stored as binary data).
- **Backend**: A Python FastAPI service running in Docker.
- **Frontend**: A React Static Site. Any request to `/api/*` is automatically routed to the backend.

## Troubleshooting

- **Database Connection**: If the backend fails to start, check the `DATABASE_URL` in the Render dashboard. It is automatically managed by the Blueprint.
- **WebAuthn Errors**: Ensure `RP_ID` and `RP_ORIGIN` exactly match your frontend URL. WebAuthn is very strict about domains.
- **Model Loading**: If you see "Loaded from database binary" in the backend logs, it means the persistence is working correctly!
